The Challenge

Why Single-Model Approaches Fail

❌ Naïve Approach

  • Send every task to the same large model
  • Waste tokens on simple math (e.g., "15% of 200")
  • No category awareness → wrong prompt → bad answers
  • No validation → refusals/truncations go undetected
  • Single failure mode → entire batch at risk

✅ Our Approach

  • 5-layer pipeline intercepts & routes each task
  • Math solved locally at 0 token cost
  • Category-tuned prompts maximize accuracy
  • Speculative validation catches errors before scoring
  • Isolated error handling — one failure won't crash the batch

Track 1 scoring: Accuracy Gate first, then rank by ascending token count. You must pass accuracy to even be ranked — then every token saved matters.

Token Optimization

Three Zero-Token Tiers

⚡ Tier 0 — Deterministic Solver
Safe AST parser evaluates arithmetic, percentages, square roots, unit conversions (30+ pairs), temperature conversions, and discount problems. 100% accuracy, 0 tokens.
🏷️ Tier 1 — Regex Heuristic Triage
20+ regex patterns match task categories at zero cost. Covers ~80% of tasks. Ambiguous cases fall back to a lightweight model call for classification only.
🧠 Local Model — Qwen 2.5 1.5B
Runs inside the Docker container via llama-cpp-python. Handles sentiment, NER, summarization, and factual tasks at 0 Fireworks tokens. Falls back to API on failure.
🗜️ Prompt Compression
Strips redundant whitespace, collapses blank lines, and cleans punctuation while preserving code block formatting. Saves ~10-15% input tokens on every API call.

Accuracy & Bonus

Validation + Gemma Strategy

✅ Speculative Validation

  • Empty/whitespace-only responses → rejected
  • Truncated text (ends with "...") → rejected
  • Refusal phrases ("I'm sorry", "As an AI") → rejected
  • NER: validates JSON structure
  • Math: checks for numeric presence
  • Code: looks for def/function/return
  • Suspicious → corrective retry with category-specific instructions

💎 Gemma 4 Bonus Strategy

  • Factual, sentiment, NER, summarization → try Gemma first
  • Circuit-breaker: single failure disables Gemma for remaining tasks
  • Time-limited: won't exceed 5s per attempt
  • Silent fallback to MiniMax M3 on any error
  • Protects 10-minute deadline
  • Eligible for $1,000 Gemma bonus prize
